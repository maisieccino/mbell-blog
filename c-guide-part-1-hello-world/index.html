C Guide Part 1/? - Hello World!

Posted on 05 October 2015

Comments

Hello! This is going to be a guide to C for people in my class, which I’ll update as we go along. Enjoy, and do let me know if there’s anything else you need a hand with :smile:

I’ll be assuming that you are using a UNIX/UNIX-like operating system and using the command line, so any code that begins with a $ symbol means that it’s meant to be typed into the command prompt (without the $!)

For this first post, I’ll go through a very simple program called ‘hello_world.c’. Open your favourite text editor (obviously vim) and enter this:

#include<stdio.h>


int main() {
  int number = 3;
  printf("Hello world!\n");
  printf("%i",number);
  return 0;
}

Now, save the file (:w helloworld.c in vim) and send the text editor to the background (press ctrl+z, you can return to it with the $ fg command if you wish). Then, to compile your program, you simply run $ gcc helloworld.c. The compiled program can then be ran by executing $ ./a.out, the file created during compilation.

You should have the words “Hello world!”, then below it “3” appear on your screen, and finally the command prompt appear below it. Success! You’ve created your first C program. Let’s have a look at how it works.

Code in C and many other similar languages is broken up into 'blocks’, and these are represented with curly braces surrounding the code (or moustache brackets as I call them). Breaking your code up into blocks means that you can reuse the same code many times without writing it again, saving you time and boredom.

Notice how the code block in our program is preceded with int main(). This means that the block is a function. If you’re unfamiliar with the term, think of it like this: Nothing or something goes in, and something comes out. A function may accept data (these are called arguments, or parameters), and if so, these will be 'declared’ by appearing in the round brackets. The word 'main’ is the name of our function. Finally, the key word ’int’ tells us that the function will return an integer variable once it is complete. This is handy as we can use this to send data back to whatever called this function.

Yep, that’s right, once you have declared a function, you can refer to it later with its name and run the same code! This is an integral part of programming, and saves you soooo much time. Inside this code block, you’ll see the second line starts off with printf(...). Printf is a function that we have called; it takes the parameter (in our case it is “Hello world!\n”) and displays it in the terminal. But where was this function declared? That’s what the first line of the program, #include<stdio.h> is for. It tells the compiler to include all of the functions and other things from the file 'stdio.h’, a file that’s all about input and output.

The first line in our program is assigning a variable. Variables are great in programming, as they allow you to store values and refer back to them. In this case, we are declaring a variable called “number” which is an integer (remember the key word from above!) and assigns it (sets the value to) 3. There are many other variable types, but we shall come back to them later. Remember though, that you must always declare a variable before you use it!

The second printf(...) statement looks a little different to the first. You’ll notice that inside the brackets, there are two things separated by commas. This is because printf can actually take many arguments, and in this case it accepts a string and an integer. The magic with printf (and another function called scanf()) is that you can type in a special sequence into the ouput, in this case “%i”, and it will use the next parameter and display it instead, as long as it is an integer. (“%i” means integer). So, in effect, printf("%i",number) simply means, print the value of number to the screen.

Finally, the return 0 means that the function should return the integer 0. As this is the main function of the program, this is handy to tell whether it ran properly or not. If a program exits but does not return 0, then clearly something has not gone well. In C, all functions must return a value, even if that value does not get used! Remember this.

So, hopefully that covers the basics for now. Tomorrow I’ll go through the reverse.c program that was shown in class today. If you want a fun challenge, try writing a program that accepts a number from the user and returns the next number! Hints:

  • scanf("%i",integer) lets you read integers as input.

  • Simple mathematical operators work in C! + for addition, - for subtraction, * for multiplication, and / for division.

  • Your program needs to return 0 to succeed!
