C Guide Part 2/? - Types and Functions and Arrays

Posted on 11 October 2015

Comments

So, in the last tutorial I explained the basics of the C language and how to write simple programs. Today I will show you how to expand on that, using different variable types, functions to structure your code, and arrays to store a lot of data very easily.

Firstly, types. Variables take up various names, forms, and storage size on memory. So perhaps it’s unsurprising to learn that there are different types of variables that are used in C. We shall go through the most common ones (summarized from this Wikipedia page (accessed on 11th October 2015):

  • int - the integer type, that is at least 16 bits in size. A whole number that is between -32767 and +32767. However, there is also the unsigned int, which allows higher numbers to be stored in the same space, but only positive numbers. You can also get long and long long, which can store larger integer values.

  • float - the floating point type. This can store very large and very small numbers with some amount of accuracy, as well as numbers with a fractional part. It is stored using in the form x * 2^y, which is the binary version of the standard form we all know and love (x * 10^y). There is also the double type, which funnily enough provides double the precision (?) by using more bits, which is good for when you are dealing with precise numbers (especially when in binary, 0.1 is an irrational number!) Finally there’s a long double for when you are feeling really precise.

  • char - the character type. char can store the value of a single text character, making it very handy when it comes to text manipulation. However, char is actually an integer type in its own right and is handy when dealing with very small integers.

  • bool - the boolean type. Also known as: “yes/no”, “true/false”, “0/1”. The value of a boolean is either true or false and is handy for logic situations. Whenever you use an if statement you are actually testing a boolean! You need to include stdbool.h in your code to use these, and in many cases you’re better off just using an integer.

Now, let’s talk about functions. Functions are a great way of breaking up your code and making it more readable, as well as reducing the number of times you have to type out your most common procedures. Functions can also be exported to be used in other programs, but we’ll come to that later on.

The important thing to remember with functions is that they have to be declared before you use them, otherwise the compiler will not be able to find them in time. int main() is the most common function you will declare, and it was explained in the last tutorial. A function is declared as so:

type functionName (type arg1, type arg2, ..) {
  // Insert your code here
  return value;
}

So, a basic function to add two numbers and return the result would look like this:

int sum(int num1, int num2) {
  return num1 + num2;
}

A little trivial perhaps, but you get the idea of how functions work at their most basic level.

Finally, let’s explore arrays a little bit. In programming, it’s a lot nicer to make lists of things we want to process, rather that assign separate variables for each one. For instance, if we were writing a program to find the sum of 10 numbers, it would be mad to create variables int num1 int num2 et cetera, right? So, instead, we can define an array (a list), like so: int nums[10]. The value inside the square brackets simply tells the compiler how big the array will be, and therefore how much memory should be given to store all the variables. Each value in the array can be ready using its ‘index’, or position, in the array. For instance, for the sixth value, we’d use nums[5]. Wait, what?! I forgot to mention, programmers start counting from 0. So the first value of the array is actually in the 0th position :blush:

Alright, so the challenge from this lesson is to try and write a program that does the insertion sort algorithm (remember Theory I?) on an array of six integers and prints out the array in the correct order. No need to use a separate function though! A hint is that you can assign values to the whole array when you declare it like so: int array[4] = { 4, 2, 3, 1 }; Good luck! :keyboard:
